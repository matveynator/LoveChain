# LoveChain TinyGO

This folder is reserved for future smart ring, pendant, and low-power device logic.

Planned responsibilities:

* proximity signals;
* simple signed presence events;
* ring-to-phone communication format;
* minimal shared LoveChain event structures;
* firmware experiments for TinyGo-supported boards.

TinyGO code should stay small, explicit, and portable. The phone apps remain responsible for full LoveBlock storage and user confirmation.
